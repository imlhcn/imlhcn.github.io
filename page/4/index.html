ssh做socks5代理

Posted on 2017-08-14 | Edited on 2019-05-01 | In 独家分享

SSH -D

首先，还是要有一台没被墙的机:10.0.0.1

主要参数说明:
7070 为sock5的端口,user对应链接的用户,
简单的就是百度 “ssh免密登录”
id_rsa这个文件是标准的openssh 的sshkey,可以工具(ssh-keygen)生成，一般是一对的(id_rsa+id_rsa.pub)
需要把 id_rsa.pub的内容(单行的)放到 10.0.0.1 的 /home/user/.ssh/authorized_keys(注意 .ssh 和authorized_keys的权限644就好，太高了连不上)
连接有问题可以看看 /var/log/secure

在浏览器设置 socks5代理

参数 localhost:7070
OK,可以利用 10.0.0.1的网络上网

应用场景

  • 某些系统有IP白名单限制，把白名单放10.0.0.1 然后本地的代理到7070即可，愉快调试了
  • 某台机只开了22(ssh)端口，上面挂了一个3306(mysql)只允许localhost访问，那么留意一下MYSQL客户端的（SSH Tunnel)这种连接方式
  • 有一台没墙的机，还没部署好ss就可以立即用这个来科学上网了

扩展

某些软件只支持 http代理，怎么破？

有 privoxy 或proxychains 可以将socks转为http

redmine + nginx + thin

yum install git vim gem ruby-devel mysql-devel ImageMagick
gem install bundler

Step 1 下载redmine

git clone https://github.com/redmine/redmine.git
cd redmine
git checkout 3.3-stable

Step 2 Mysql

Step 3 配置

cp config/database.yml.example config/database.yml
vim config/database.yml

cp config/configuration.yml.example config/configuration.yml
vim config/configuration.yml

Step 4 依赖

vim Gemfile 改为国内ruby源

bundle install –without development test

bundle install –without development test rmagick

Step 5 生成token

bundle exec bin/rake generate_secret_token

Step 6　数据库见表

RAILS_ENV=production bundle exec bin/rake db:migrate

Step 7 默认数据

RAILS_ENV=production bundle exec bin/rake redmine:load_default_dat

Step 8 文件权限

mkdir -p tmp tmp/pdf public/plugin_assets
sudo chown -R redmine:redmine files log tmp public/plugin_assets
sudo chmod -R 755 files log tmp public/plugin_assets

Step 9 测试安装

bundle exec bin/rails server -b 0.0.0.0 webrick -e production
