ssh做socks5代理

发表于 2017-08-14 | 分类于 独家分享

SSH -D

首先，还是要有一台没被墙的机:10.0.0.1

主要参数说明:
7070 为sock5的端口,user对应链接的用户,
简单的就是百度 “ssh免密登录”
id_rsa这个文件是标准的openssh 的sshkey,可以工具(ssh-keygen)生成，一般是一对的(id_rsa+id_rsa.pub)
需要把 id_rsa.pub的内容(单行的)放到 10.0.0.1 的 /home/user/.ssh/authorized_keys(注意 .ssh 和authorized_keys的权限644就好，太高了连不上)
连接有问题可以看看 /var/log/secure

在浏览器设置 socks5代理

参数 localhost:7070
OK,可以利用 10.0.0.1的网络上网

应用场景

  • 某些系统有IP白名单限制，把白名单放10.0.0.1 然后本地的代理到7070即可，愉快调试了
  • 某台机只开了22(ssh)端口，上面挂了一个3306(mysql)只允许localhost访问，那么留意一下MYSQL客户端的（SSH Tunnel)这种连接方式
  • 有一台没墙的机，还没部署好ss就可以立即用这个来翻墙了

扩展

某些软件只支持 http代理，怎么破？

有 privoxy 或proxychains 可以将socks转为http

说明：
1.server_name 如果有域名，可以用域名
2./mysite/html下面则是静态页根目录
3./api 要求 localhost 上需要有后台的web服务开启在8088端口，而且，所有接口必须以 /api开头

原理

其实就是nginx的静态服务+反向代理

静态文件
反向代理
什么是反向代理?

nginx的前后端分离，要求 前后端，必须要一某种明确的界限去拆分，可能可以根据（ContentType 例如 text/plain css 等 作为前端， application/json作为后端）
但是，实战经验上看，最直观、稳定的，还是 用url前缀。比如：
一： /html -> 前端 /api -> 后端
二： /html -> 前端 / -> 后端
三: / -> 前端 /api/ ->后端
因为一般域名访问(如:http://www.baidu.com)
都是直接前端（一般上百度不会 http://www.baidu.com/home这么啰嗦)，
所以采取了第三种（见上述配置)

实战

配置文件演示了，一个典型的系统：
前台+后台 都是前后端分离
前台:可能是一个移动优先的h5单页应用，前端可能使用vue,后端可能能使用 node+koa2+java?
域名为 www.mysite.com
后台：是一个运营者、管理者使用的基于PC浏览器的后台系统，前端可能用angular或react,后端 Java SpringMVC
域名为 admin.mysite.com

注意 以下元素，可以根据实际需要任意组合
1.静态文件路径
2.proxy_pass端口
3.proxy_pass URL
4.location 前缀
